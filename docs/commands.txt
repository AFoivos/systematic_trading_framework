# Φύλλο εντολών του Systematic Trading Framework

Τελευταία ενημέρωση: 2026-07-11

Οι εντολές εκτελούνται από το repository root, εκτός αν αναφέρεται διαφορετικά.
Τα commands είναι παραδείγματα χρήσης και δεν αναιρούν τα safety gates των
configs. Πριν από πείραμα έλεγξε dataset paths, κόστος, split semantics και
output directory.

## 1. Περιβάλλον και tests

Linux/macOS:

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Προαιρετικές tsfresh dependencies:

```bash
python -m pip install -r requirements.tsfresh.txt
```

Tests:

```bash
python -m pytest -q
python -m pytest -q tests/features
python -m pytest -q tests/signals
python -m pytest -q tests/execution
python -m pytest -q tests/market_making tests/market_data
python -m pytest -q apps/trading_dashboard/backend/app_tests
python -m pytest -q mcp_server/tests
```

Με Docker:

```bash
docker compose build
docker compose run --rm app pytest -q
```

## 2. Βασική εκτέλεση experiment

Το ασφαλές, ελάχιστο παράδειγμα τεκμηρίωσης είναι:

```bash
python -m src.experiments.runner docs/examples/minimal_experiment.yaml
```

Το αντίστοιχο Docker command:

```bash
docker compose run --rm app python -m src.experiments.runner config/experiments/foundation_alpha/ethusd_30m_trial_0041_alpha_lab/05_signal_lab/ethusd_30m_trial0041_sig_dense_net_v1.yaml

Το οπτικό feature/signal/target lab:

```bash
python -m src.experiments.runner config/experiments/lab/feature_signal_target_lab.yaml
```

Αντιπροσωπευτικά ενεργά configs:

```bash
python -m src.experiments.runner config/experiments/London_NY/raw/xauusd_london_orb_breakout_raw_v3.yaml
python -m src.experiments.runner config/experiments/ehlers/us100_30m_ehlers_decycler_continuation_long_v1.yaml
python -m src.experiments.runner config/experiments/foundation_alpha/usoil/usoil_30m_lightgbm_h24_structured_tail_alpha_v1.yaml
```

Τα παραπάνω απαιτούν τα datasets που δηλώνει κάθε YAML. Ένα υπαρκτό config δεν
εγγυάται ότι το αντίστοιχο local dataset ή η optional model dependency είναι
διαθέσιμα.

## 3. Optuna

Ελαφρύτερη smoke προδιαγραφή:

```bash
python -m src.experiments.optuna_search \
  config/optuna/ehlers/optuna_ehlers_ml_long_lightgbm_30m_v1_smoke.yaml
```

Overrides:

```bash
python -m src.experiments.optuna_search \
  config/optuna/ehlers/optuna_us100_30m_ehlers_decycler_continuation_long_v1.yaml \
  --n-trials 20 \
  --n-jobs 1 \
  --sampler random \
  --seed 42
```

Resume σε SQLite:

```bash
python -m src.experiments.optuna_search \
  config/optuna/ehlers/optuna_us100_30m_ehlers_decycler_continuation_long_v1.yaml \
  --storage sqlite:///logs/optuna/us100_ehlers.db \
  --study-name us100_ehlers \
  --load-if-exists
```

Η Optuna εκτέλεση μπορεί να είναι βαριά. Μην χρησιμοποιείς το untouched test
subset για επιλογή trials.

## 4. Data preparation

Οι download εντολές κάνουν εξωτερικές network κλήσεις:

```bash
FROM_DATE=2024-01-01 TO_DATE=2024-02-01 \
  bash scripts/download_dukascopy_30m.sh

START_YEAR=2020 END_DATE=2026-01-01 \
  bash scripts/download_dukascopy_30m_quarterly.sh
```

Συγχώνευση και παραγωγή mid-price αρχείων:

```bash
python scripts/merge_dukascopy_quarterly_30m.py \
  --input-root data/raw/dukascopy_quarterly \
  --output-dir data/raw/dukascopy_30m

python scripts/prepare_dukascopy_30m_bid_ask_mid.py \
  --input-dir data/raw/dukascopy_30m \
  --output-dir data/raw/dukascopy_30m_clean
```

Οι μετατροπές δεδομένων πρέπει να διατηρούν timestamps, timezone, bid/ask
προέλευση και snapshot provenance.

## 6. MT5 dry-run

Η ακόλουθη εντολή επιβάλλει `--dry-run` και επεξεργάζεται μία κλεισμένη μπάρα:

```bash
python scripts/run_mt5_demo_bot.py \
  --config config/execution/mt5_demo.yaml \
  --once \
  --dry-run
```

Μην αφαιρείς το `--dry-run` χωρίς ανεξάρτητο έλεγχο λογαριασμού demo, symbol
mapping, risk limits, kill switch και `order_send` gates.

## 7. Market-making data και paper replay

Ο public Kraken Spot collector κάνει network κλήση αλλά δεν χρησιμοποιεί
credentials και δεν στέλνει orders:

```bash
python scripts/collect_kraken_orderbook.py \
  --config config/execution/kraken_futures_demo_market_making.yaml \
  --output logs/experiments/market_making/orderbook_events.csv
```

Replay του αποθηκευμένου CSV:

```bash
python scripts/run_market_making_paper.py \
  --config config/execution/kraken_futures_demo_market_making.yaml \
  --input-events logs/experiments/market_making/orderbook_events.csv \
  --timestamped-output
```

Το synthetic paper mode χωρίς `--input-events` απαιτεί διαφορετικό, ελεγμένο
config με `execution.mode: paper`. Το tracked config παραπάνω είναι
`data_only`.

Diagnostics:

```bash
python scripts/analyze_market_making_run.py \
  --latest \
  --reports-root logs/experiments
```

## 8. MOMENT research

Το άμεσο config χρησιμοποιεί deterministic fixture από προεπιλογή και απαιτεί
ήδη υπάρχοντα event files:

```bash
python scripts/run_market_making_moment_experiment.py \
  --config config/experiments/market_making/market_making_moment.yaml
```

Η staged ροή παρακάτω ξεκινά public collection 100.000 events και μπορεί να
χρησιμοποιήσει βαρύ MOMENT backend:

```bash
python scripts/run_market_making_pipeline.py \
  --config config/experiments/market_making/market_making_moment_collect_100k_pipeline.yaml
```

Τα MOMENT metrics είναι ερευνητικά diagnostics. Δεν αποτελούν executable OOS
PnL ή live evidence.

Το Kraken Futures demo adapter είναι scaffold: δεν υπάρχει tracked config με
`execution.mode: kraken_futures_demo` και `allow_demo_orders: true`, ενώ η REST
τοποθέτηση/ακύρωση orders δεν είναι υλοποιημένη. Μην χρησιμοποιείς το
`run_kraken_futures_demo_mm.py` ως trading command.
